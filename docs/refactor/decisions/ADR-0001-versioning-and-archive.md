# ADR-0001 版本归档和重新立项

日期：2026-07-01

## 状态

accepted

## 背景

当前 JUGO 项目尚未正式上线，但需求发生大幅变化，后续工作不适合在旧需求和旧实现上继续小修小补。需要把当前资料固定为一个归档版本，同时给重构项目建立新的版本起点。

## 决策

- 当前未上线项目归档为 `v0.1.0-legacy`。
- 重构项目重新立项为 `v0.2.0-planning`。
- `v0.1.0-legacy` 只作为资料归档和参考，不承接新需求。
- `v0.2.0-planning` 从需求分析开始，不默认继承旧功能、旧接口和旧实现。
- 暂不使用 `v1.0.0`，因为新项目尚未达到正式发布状态。

## 后果

- 旧项目资料有明确归属，后续不会和新需求混写。
- 新项目可以重新定义产品范围、领域模型、API 和架构。
- 旧代码是否复用需要逐项评估，不能默认沿用。
- 后续正式可发布版本仍可保留 `v1.0.0` 语义。
