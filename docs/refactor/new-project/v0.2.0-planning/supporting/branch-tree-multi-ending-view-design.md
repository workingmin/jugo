# Web端「分支树全局多结局概览视图」页面设计方案（对标 Narratium，适配V1\.0负责人顶层评审流程、Web轻量化、全模块数据互通）

## 一、页面基础定位

1. 载体：纯 Web 页面，宽屏自适应，专为**多分支 / 多结局互动剧本、游戏叙事**顶层全局评审设计；

2. 对标基准：Narratium 树形分叉叙事画布、分支条件、多结局收敛链路、分支任务分配体系；

3. 核心差异化（弥补 Narratium 原生短板）

    - 与世界观、矩阵时序、节拍板**一套底层数据双向互通**；

    - 内置全局设定冲突校验，联动世界观库检测人设 / 时序矛盾；

    - 支持一键筛选单分支，自动跳转对应矩阵时序视图；

    - 配套团队任务分配、负责人评审批注体系；

    - 打通 AI 全局生成链路，全分支统一注入世界观约束；

4. 固定评审链路（V1\.0 标准）
世界观知识库 → 本分支树视图全局概览分叉与结局 → 选中单分支自动过滤进入矩阵时序 → 时序节点跳转节拍板单场核查

## 二、页面整体布局（四栏 Web 分层布局）

### 顶部全局操作栏（统一产品导航，固定置顶）

1. 视图快速切换组：世界观 / 矩阵时序 / 分支树 / 节拍板

2. 分支筛选 \& 任务管控区（负责人核心工具）

    - 分支下拉筛选：全部分支 / 单一结局分支（选中后中央画布隔离其他线路，时序视图同步过滤）

    - 负责人任务标签：按编剧组员筛选分支，快速查看对应人负责线路

3. 画布通用操作工具

    - 缩放滑块 40%\~200%、画布拖拽平移、自动规整树形布局

    - 全局叙事校验：扫描分支逻辑、时序断层、伏笔遗漏、人设冲突，输出验收报告

    - 批量导出：全分支叙事大纲 / 单分支时序大纲

4. 画布配置

    - 新增分叉节点、新增结局终点、自定义分支线条颜色

    - 显示开关：分支条件弹窗、人物标签、批注红点、收敛节点标记

5. 引导提示：多分支项目专属分步评审指引

### 左侧：分支层级 \& 任务侧边栏（树形目录，对标 Narratium 分支管理）

1. 顶层根节点：主线基础剧情（所有分叉的统一前置基底）

2. 二级：分叉节点分组（关键选择点位）

3. 三级：独立分支线路，每条线路绑定：

    - 自定义线条颜色（时序、节拍卡片同步同色标签）

    - 负责人 / 编剧分配标签（团队多人分工标识）

    - 结局类型标记：好结局 / 坏结局 / 隐藏结局 / 中立结局

4. 单分支右键操作：

    - 【筛选本条分支】中央画布只展示该线路；

    - 【跳转矩阵时序】一键切换时序视图，自动过滤当前分支内容；

    - 分配负责人、重命名、复制分支、删除分支；

5. 底部快捷操作：新建分叉节点、批量复制主线分支、一键同步世界观约束至全分支。

### 中央核心画布区（对标 Narratium 树形分叉核心交互）

#### 底层画布逻辑

1. 起点：唯一主线根节点（对应矩阵时序公共前置剧情，所有分支共享）；

2. 分叉节点：同一剧情点位生成多条**互斥选择线路**，玩家 / 读者只能选择其一；

3. 收敛节点：多条分支重新汇合，回归公共主线时序；

4. 终点：独立结局节点，每条分支最终绑定专属结局；

5. 线条规则：线条颜色匹配分支侧边栏配色，清晰区分不同结局线路。

#### 分支节点三大类型（画布核心元素）

1. **主线公共节点**
所有分支必经剧情，无选择分叉；自动同步矩阵时序公共轨道卡片，双击可跳转时序 / 节拍板。
标识：实心矩形，底色统一灰色。

2. **分叉选择节点**
故事关键抉择点，拆分多条互斥分支；可填写选择触发条件、剧情变量（信任值、阵营好感等）。
标识：菱形，边角带选择标记。

3. **结局终点节点**
单条分支最终收尾，标记结局类型；可绑定结局专属世界观人物状态、势力走向。
标识：圆角圆形，按结局类型填充区分颜色（绿 = 好结局、红 = 坏结局、紫 = 隐藏结局）。

#### 节点统一交互（适配负责人评审）

1. 拖拽：拖动分叉、终点调整树形布局，横轴坐标自动同步矩阵时序时间刻度；

2. 双击节点：弹出编辑面板，修改分支名称、选择条件、绑定出场人物 / 伏笔；

3. 右键菜单（评审高频）
① 筛选本条分支：画布隔离其他线路；
② 跳转矩阵时序：自动过滤对应分支时序轨道；
③ 跳转世界观：查看绑定人物 / 势力设定；
④ 添加评审批注：红点同步至时序、节拍板同节点；
⑤ AI 生成分支完整剧情：基于世界观 \+ 分支条件生成整套分支剧本；

4. 辅助视觉标记

    - 批注红点：存在修改意见的节点右上角红点；

    - 冲突黄标：全局校验检测出设定 / 时序矛盾的节点；

    - 伏笔虚线：绑定全局伏笔库的节点底部虚线标识。

### 右侧：分支详情编辑面板（选中节点自动弹出）

1. 基础信息：分支名称、分支类型、归属编剧、结局分类、线条配色；

2. 分支触发条件编辑：剧情变量、前置行为约束（适配游戏 flag 机制）；

3. 关联资源栏

    - 出场人物 / 势力列表：点击一键跳转世界观知识库；

    - 绑定伏笔：新增 / 解绑全局伏笔条目；

    - 关联时序区间：显示该分支覆盖的全局时间轴范围；

4. 评审批注列表：所有修改意见，支持回复、标记已解决；

5. 核心快捷按钮：
【筛选单分支】【跳转矩阵时序视图】【AI 扩写全分支剧情】【全局校验本条分支冲突】

## 三、线性项目 / 多分支项目 视图差异化控制

1. **线性单结局项目（网文 / 影视 / 短剧模板）**
顶部导航隐藏「分支树」入口，侧边栏、分叉节点、分支筛选全部屏蔽，无任何分叉相关 UI，避免新手干扰。

2. **多分支互动 / 游戏项目（模板 2/3/4）**
完整开放全部分支画布功能，支持顶层架构规划、任务拆分、单分支过滤、逐层下钻评审。

## 四、四大核心数据互通联动（Narratium 原生不具备，V1\.0 硬性需求）

### 1\. 与矩阵时序视图双向联动

1. 分支树选中任意分支 → 矩阵时序自动过滤，仅展示该分支专属轨道卡片，其余分支灰化隐藏；

2. 矩阵时序分叉节点卡片，可一键跳转分支树对应分叉点位；

3. 拖拽分支树节点调整横轴时间，时序画布对应卡片同步平移更新时间刻度。

### 2\. 与节拍板双向联动

1. 分支筛选后，节拍画布自动同步过滤仅展示当前分支场景；

2. 分支终点、分叉节点可批量生成对应节拍卡片，同步写入时序视图。

### 3\. 与世界观知识库全局绑定

1. 所有分支、分叉、结局节点绑定的人物 / 势力 / 伏笔，实时同步至知识库条目；

2. 点击人物标签直接跳转世界观详情页；

3. 全局校验自动扫描全分支人物行为是否违背世界观设定，标记冲突节点。

### 4\. AI 全分支联动生成

1. 单分支 AI 生成：读取当前分支条件 \+ 全套世界观，生成该分支完整时序 \+ 节拍剧本；

2. 全项目一键生成：聚合所有分支、主线、世界观数据，输出完整多结局小说 / 游戏剧本。

## 五、面向项目负责人评审的专属交互设计

1. 标准顶层下钻链路闭环：
完善世界观 → 分支树全局查看所有分叉、结局分布、分支体量均衡度 → 筛选单分支跳转矩阵时序查看完整并行支线 → 时序异常节点跳转节拍板核查单场细节；

2. 全视图批注同步：分支树添加的修改意见，在时序、节拍板对应节点统一显示红点；

3. 批量任务拆分：总编剧在分支树直接给每条分支分配组员，组员登录后仅可见自身负责线路；

4. 全局叙事一键体检：自动识别分支断层、结局体量失衡、伏笔只埋不收、人物长期掉线；

5. 只读评审模式：负责人锁定画布编辑权限，仅可添加批注，防止底层分支架构被误修改。

## 六、对比 Narratium 原生产品核心差异化

### Narratium 原生短板

1. 轻量独立工具，无配套时序、节拍、世界观完整体系，数据割裂；

2. 无 Web 在线多人协同，不支持团队任务分配、分层评审；

3. 缺少全局设定冲突校验，多分支极易出现人设、时序崩坏；

4. 无完整 AI 生成链路，仅提供基础分支画布，无法一键产出完整剧本；

5. 无分层下钻评审链路，不能一键跳转时序、细化场景。

### 本 Web 分支树视图优化亮点

1. Web 端在线协作，负责人远程统筹全部分支架构；

2. 统一底层数据打通世界观、矩阵时序、节拍板，三视图联动流转；

3. 专为游戏 / 互动剧本团队评审设计，内置分支分配、全局校验验收工具；

4. 完整 AI 创作链路，顶层分支框架直接驱动完整剧本生成；

5. 严格匹配「自上而下顶层概览→单分支时序→单场节拍」标准化评审流程。

## 七、Web 端交互 \& 权限规范

1. 画布拖拽、缩放、树形布局逻辑复刻 Narratium，降低游戏编剧学习成本；

2. 所有编辑自动云端实时保存，无手动保存按钮；

3. 双视觉主题：浅色办公 / 深色夜间创作；

4. 分级权限管控

    - 项目负责人：可编辑全部分支、分配任务、全局校验、锁定画布；

    - 普通编剧：仅查看 / 编辑分配给自己的分支，其他分支只读不可修改；

5. 全局检索：顶部搜索框检索分支名称、人物、伏笔，画布自动定位对应分叉节点。
