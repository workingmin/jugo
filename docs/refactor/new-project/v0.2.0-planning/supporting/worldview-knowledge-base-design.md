# Web端「世界观知识库」页面完整设计方案（对标Campfire Write，适配V1\.0双链路剧本画布，Web轻量化实现）

## 一、页面基础定位

1. **产品载体**：纯 Web 网页，响应式，支持 PC 编剧 / 负责人主力操作，平板兼容；

2. **对标参考**：Campfire Write 结构化百科，但弱化复杂图谱渲染，强化**与三套画布实时联动、AI 全局注入**核心差异化；

3. **核心作用**：全项目统一底层约束，所有矩阵时序 / 分支树 / 节拍板共享一套设定，AI 生成剧本自动读取库内数据规避 OOC、时序冲突；

4. **访问入口**：顶部全局导航固定「世界观」Tab，任意画布视图均可一键跳转，画布卡片支持右键 “跳转对应世界观条目”。

## 二、页面整体布局（三栏经典 Web 布局，贴合 Campfire 交互逻辑）

### 左侧：分类导航侧边栏（固定悬浮，永久可见）

树形分类目录，支持自定义文件夹，内置 6 套系统默认分类（不可删除，可新增子项）

1. **人物档案【核心高频】**

    - 子分类：主角、配角、反派、NPC、特殊角色、家族 / 血脉分组

    - 每条人物卡片字段：基础人设、外貌、性格、动机、过往剧情、羁绊关系、出场时序、人物弧光记录、专属标签

2. **势力 / 阵营**

    - 国家、宗门、公司、虫族 / 人族阵营、同盟 / 敌对关系、内部架构、资源掌控、核心目标（适配 RTS 游戏、玄幻网文）

3. **场景 / 地点**
城市、秘境、战场、关键房间、特殊空间、场景专属规则、常驻人物

4. **世界基础规则**
历法、能量体系（灵力 / 泰伯利亚 / 灵能）、禁忌、时间规则、物理设定、魔法 / 科技体系

5. **伏笔 \& 线索库（联动画布核心）**
全局伏笔登记，可绑定画布时序 / 分支节点，自动标记在哪条剧情线回收伏笔

6. **道具 / 特殊物品**
信物、武器、秘宝、关键道具、持有角色、触发剧情条件

7. 自定义文件夹（用户自建，如种族、神话、历史事件）

### 中间：条目列表面板

1. 选中左侧分类后，中间展示该分类下全部条目卡片；

2. 列表支持：搜索、标签筛选、按创建时间 / 出场时序排序；

3. 卡片极简预览：名称 \+ 关键标签 \+ 简介片段，鼠标悬浮预览完整摘要；

4. 操作按钮：新建条目、批量导入、批量绑定画布节点、批量同步 AI 缓存。

### 右侧：条目详情编辑面板（核心编辑区）

选中中间列表任意条目，右侧加载完整表单，支持富文本、图片上传、标签绑定、关联其他世界观条目；
编辑实时自动保存，无手动保存按钮。

## 三、核心模块详细字段设计（Web 表单轻量化，兼顾专业度，不做冗余图谱）

### 模块 1：人物档案（最高频使用）

1. 基础信息：姓名、称号、年龄、种族、所属势力、性别、外貌配图上传、标志性特征

2. 性格 \& 内核：表层性格、深层欲望、内心创伤、恐惧、价值观、行为逻辑

3. 人物关系：快捷关联其他人物（父子 / 仇敌 / 恋人 / 上下级），自动生成关系标签

4. 剧情轨迹：出场章节 / 时序区间、关键成长节点、死亡 / 结局归属（绑定对应分支树结局）

5. 专属约束标签：AI 生成强制遵守（如 “该角色绝不杀人”“厌恶谎言”）

6. 关联剧情：自动展示所有画布中出现该人物的时序卡片、分支节点、节拍场景，一键跳转对应画布位置

### 模块 2：势力 / 阵营

1. 基础：名称、领袖、管辖区域、核心资源、创立背景

2. 对外关系：敌对 / 同盟 / 中立势力快捷绑定

3. 内部派系：分支部队、核心人物列表

4. 阵营目标：短期计划、终极诉求、标志性行动逻辑

5. 关联剧情：自动带出所有涉及该势力的剧情节点

### 模块 3：伏笔线索库（画布强联动核心）

1. 伏笔名称、初次出现剧情节点（自动绑定矩阵时序卡片）

2. 伏笔类型：道具伏笔、人物秘密、时空伏笔、反转伏笔

3. 回收节点：下拉选择分支树 / 时序节点，标记在哪条线完成回收

4. 约束提示：AI 生成时必须完成伏笔回收，禁止遗忘

### 模块 4：世界规则

纯结构化文本 \+ 标签，分为时间、能量、空间、社会四大子面板，每条规则可开启「AI 强制校验」开关；开启后 AI 写剧本违反规则时，系统弹窗提示冲突。

## 四、Web 页面专属联动能力（区别 Campfire Write，产品核心差异化）

### 1\. 双向联动画布（V1\.0 硬性需求）

1. 从世界观条目 → 跳转画布
任意人物 / 势力 / 伏笔详情页，底部「关联剧情」列表，点击条目直接切到矩阵时序 / 分支树 / 节拍板对应视图，自动定位卡片；

2. 从画布 → 跳转世界观
矩阵时序、分支树、节拍板任意卡片内选中人物 / 地点 / 伏笔，右键菜单「查看世界观档案」，新标签页打开知识库对应条目。

### 2\. 全局 AI 上下文同步（核心功能）

页面顶部全局开关：【同步全部世界观至 AI 上下文】

- 开启后，所有时序、分支、节拍卡片一键生成剧本时，自动拉取整套知识库数据作为前置 Prompt；

- 支持单条条目单独开关：如 “次要 NPC 不注入 AI”，精简上下文长度，提升生成速度。

### 3\. 冲突校验联动（适配负责人验收流程）

负责人在知识库页面点击【全局设定校验】：
系统扫描全部分支、时序、节拍内容，自动检索：
① 人物行为违背自身设定；② 势力关系逻辑矛盾；③ 伏笔未回收；④ 时空时序冲突；
生成校验报告，每条冲突附带「跳转剧情节点」按钮，直达画布修改。

## 五、页面顶部全局操作栏（Web 顶部导航）

1. 项目名称下拉：切换项目（多项目管理）

2. 【同步至画布】：手动刷新知识库数据，同步到所有视图缓存

3. 【全局 AI 校验】：一键生成设定冲突报告（负责人验收核心按钮）

4. 【批量导入 / 导出】：JSON 模板导入导出，兼容本地备份、团队分发

5. 视图快速跳转快捷按钮：矩阵时序 / 分支树 / 节拍板（无需切顶部大导航）

6. 帮助指引：新手分步引导弹窗（线性模板 / 多分支模板差异化指引）

## 六、适配两套创作 / 评审流程的交互设计

### 流程 1：线性单结局项目（无分支树）

1. 创作者 / 负责人进入「世界观」完善全部设定；

2. 顶部快捷按钮一键跳转矩阵时序视图，开始搭建全局轨道；

3. 时序卡片可随时右键打开对应人物 / 地点档案，随时核对设定。

### 流程 2：多分支多结局项目

1. 总负责人先完整搭建世界观知识库，统一全分支底层约束；

2. 顶部快捷按钮跳转分支树，概览所有多结局线路；

3. 选中单分支跳转矩阵时序，节点下钻节拍板；

4. 评审时发现人设 / 设定矛盾，一键切回知识库修改，修改后全画布实时同步更新。

## 七、和 Campfire Write 核心差异（Web 产品优势）

1. Campfire 短板：知识库与剧情画布完全割裂，无自动联动，AI 无全局注入能力；

2. 本产品 Web 端优化点：

    - 知识库、三套画布**一套底层数据互通，实时双向同步**；

    - 内置 AI 全局约束能力，Campfire 无原生 AI；

    - 轻量化 Web，无需客户端安装，团队多人在线同时查看知识库；

    - 简化复杂 3D 关系图谱（V1\.0 不做），仅保留文字 \+ 标签关联，降低开发成本；

    - 贴合评审流程，内置全局设定校验工具，适配项目负责人验收场景。

## 八、页面视觉 \& 交互规范（Web 通用）

1. 风格：简约专业编剧工具风，低饱和度深色 / 浅色双模式切换；

2. 操作：全部修改自动保存，无丢失风险；

3. 权限控制（团队模式）：

    - 负责人：完整编辑、全局校验、批量操作；

    - 普通编剧：仅查看知识库，仅可编辑自己负责分支关联条目；

4. 搜索全局检索：顶部全局搜索框，可同时检索知识库、时序卡片、分支节点、节拍文本，跨模块一键定位。
