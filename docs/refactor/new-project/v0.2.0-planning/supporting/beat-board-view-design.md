# Web端「节拍板视图」页面设计方案（对标 Final Draft Beat Board，适配V1\.0时序下钻评审链路、Web轻量化、全模块数据互通）

## 一、页面基础定位

1. 载体：纯 Web 页面，承接**矩阵时序卡片一键跳转**的细节编辑 / 评审层，是最小剧情单元编辑载体；

2. 对标基准：Final Draft Beat Board 软木板卡片、三幕式分区、场景与大纲强绑定逻辑；

3. 产品差异化升级（弥补 Final Draft 短板）

    - 和矩阵时序、分支树、世界观实时双向同步；

    - 内置 AI 单卡生成 / 全本生成入口；

    - 适配负责人自上而下评审批注流程；

    - 支持多分支标签区分，自动关联对应分支时序；

4. 标准跳转链路
线性项目：矩阵时序卡片 → 本节拍板
多分支项目：分支树筛选单分支 → 矩阵时序 → 选中节点跳转节拍板

## 二、页面整体布局（三栏 Web 分层结构）

### 顶部全局操作栏（固定置顶，统一产品导航）

1. 视图快速切换组：世界观 / 矩阵时序 / 分支树 / 节拍板

2. 上下文信息栏（跳转自动携带，负责人评审核心标识）

    - 显示来源：【来自 XX 分支 \- XX 时序轨道 \- 第 X 章节节点】

    - 标签：出场人物、所属势力、伏笔绑定标记

3. 画布通用工具

    - 缩放滑块 50%\~180%、画布拖拽平移

    - 按幕分区开关（三幕 / 四幕结构分区，对标 Final Draft 标准节拍模板）

    - 全局校验：单场景人设 / 设定冲突快速检测

    - AI 工具箱：单卡扩写、优化桥段、生成完整本场对白

4. 批量操作

    - 批量同步至矩阵时序、批量复制至其他分支、批量导出场景文本

5. 画布设置：卡片尺寸、配色跟随所属时序轨道色、显示 / 隐藏标签、批注红点显隐

### 左侧：幕结构分区侧边栏（对标 Final Draft Story Map）

1. 系统默认三幕分区：第一幕铺垫 / 第二幕对抗 / 第三幕结局，可自定义新增幕 / 序列分区

2. 分区功能：

    - 拖拽调整幕长区间，自动同步回矩阵时序横轴刻度；

    - 内置行业节拍模板：救猫咪 15 节拍、八序列结构，一键批量生成标准节拍卡片；

    - 分区颜色自定义，卡片自动继承分区底色；

3. 筛选控件：按分支 / 人物 / 势力筛选当前画布内节拍卡片，评审时快速过滤无关内容。

### 中央核心画布区（复刻 Final Draft Beat Board 软木板卡片交互）

#### 底层布局逻辑

横向线性排布，严格匹配矩阵时序横轴时间顺序；卡片左右拖拽调整顺序，同步回上层时序视图坐标；纵向可自由堆叠，不区分并行轨道（仅聚焦单场细节，并行多线在上层矩阵时序管控）。

#### 节拍卡片（核心交互单元，与时序卡片一一绑定）

1. 卡片基础字段（自动同步矩阵时序缩略内容）

    - 标题：本场核心事件；

    - 梗概富文本框：细化场景冲突、人物动作、情绪、反转、钩子；

    - 标签栏：出场人物、势力、伏笔、归属分支、幕分区；

    - 视觉标识：轨道同色色块、分叉节点三角标记、伏笔虚线标识、批注红点标记。

2. 卡片交互（贴合评审流程）

    - 双击卡片：直接编辑梗概、补充桥段细节；

    - 拖拽左右：调整剧情先后，上层矩阵时序同步变更横轴位置；

    - 右键菜单（负责人高频操作）
    ① 返回矩阵时序：一键回到上层全局视图
    ② 跳转世界观：点击人物 / 伏笔直接打开对应档案
    ③ 添加评审批注：批注同步至时序、分支树同节点，全视图红点标记
    ④ AI 扩写本场：基于世界观约束生成完整场景剧本
    ⑤ 复制至其他分支、删除、拆分多张小节拍卡片

3. 分区辅助线
垂直虚线分割三幕结构，直观查看节拍分布是否失衡，对标 Final Draft 故事节奏校验功能。

### 右侧：卡片详情 \& 批注面板（选中卡片自动弹出）

1. 完整梗概编辑区（比卡片预览完整，支持图文、分段）；

2. 关联资源栏

    - 出场人物列表：点击跳转世界观；

    - 绑定伏笔：新增 / 解绑全局伏笔库条目；

    - 归属分支：多分支项目展示该场景属于哪几条结局线；

3. 批注全记录：所有负责人、编剧修改意见，支持回复、标记已解决；

4. 快捷操作按钮
【返回上层时序】【AI 生成本场完整剧本】【校验设定冲突】【同步更新至矩阵时序】

## 三、线性项目 / 多分支项目 视图差异化控制

### 1\. 线性单结局项目

- 隐藏分支筛选、分支归属标签、分叉三角标记；

- 节拍卡片仅绑定唯一主线时序轨道；

- 流程：矩阵时序节点跳转节拍板，仅编辑单线性场景。

### 2\. 多分支互动 / 游戏项目

- 顶部开启分支筛选；

- 卡片展示归属分支标签、分叉三角标记；

- 从分支树过滤单分支后跳转节拍板，画布仅展示该分支专属节拍卡片；

- 支持右键复制当前场景至其他结局分支，快速复用桥段。

## 四、核心数据互通联动（超越原生 Final Draft，V1\.0 硬性需求）

### 1\. 与矩阵时序双向实时同步

1. 时序卡片双击跳转节拍板；节拍板修改梗概、调整顺序，上层时序缩略文字、横轴坐标实时刷新；

2. 批量新增 / 删除节拍卡片，自动同步矩阵时序新增 / 删除对应剧情节点；

3. 节拍卡片绑定人物 / 伏笔，自动同步标签至时序对应节点。

### 2\. 与分支树视图联动

1. 分叉节点节拍卡片，悬浮提示跳转分支树对应分叉点位；

2. 分支树切换筛选分支，节拍画布自动过滤仅展示该分支内容；

3. 新增分支后，可批量复制主线节拍至新分支做变体修改。

### 3\. 与世界观知识库双向联动

1. 卡片内人物、势力、伏笔标签一键跳转知识库；

2. 知识库修改人设 / 规则，节拍板全局校验，标记冲突卡片；

3. AI 生成场景时自动拉取整套世界观约束，规避 OOC。

### 4\. AI 一体化能力（Final Draft 无完整链路）

1. 单卡 AI 扩写：基于当前节拍梗概 \+ 全局世界观，生成完整单场剧本；

2. 全画布一键生成：当前分支 / 全剧所有节拍聚合，输出完整小说 / 剧本；

3. 节奏诊断 AI：扫描整张节拍板，提示中段疲软、转折点间距失衡。

## 五、面向项目负责人评审的专属交互设计

1. 固定下钻闭环：矩阵时序定位异常节点 → 跳转节拍板查看完整场景细节；

2. 全视图批注同步：节拍添加的修改意见，在时序、分支树对应节点统一显示红点；

3. 单场景快速校验：一键检测本场人物行为、势力关系、伏笔逻辑是否违背世界观设定；

4. 幕分区全局预览：快速判断三幕节拍分配是否均匀，商业结构是否达标；

5. 只读评审模式：负责人可锁定画布编辑权限，仅添加批注，防止组员误改结构。

## 六、对比 Final Draft Beat Board 核心差异化

### Final Draft 原生短板

1. 仅桌面客户端，无 Web 在线协作；

2. 节拍板与大纲、人物设定割裂，无自动冲突校验；

3. AI 仅做结构批注，无法批量生成完整场景剧本；

4. 无分支叙事联动，不支持多结局剧本分层管理；

5. 节拍板与时序画布无分层下钻链路，无法宏观→细节逐层评审。

### 本 Web 节拍板优化亮点

1. Web 在线多人协同，负责人远程实时评审各组员节拍；

2. 一套底层数据连通世界观、矩阵时序、分支树，全链路联动；

3. 完整 AI 生成链路，节拍骨架直接产出完整剧本；

4. 适配多分支游戏 / 互动剧本，支持分支筛选、分支复制；

5. 严格匹配「负责人自上而下评审」的固定下钻流程，结构化分层校验。

## 七、Web 交互与权限规范

1. 操作逻辑复刻 Final Draft 卡片拖拽、分幕布局，降低专业编剧学习成本；

2. 所有编辑自动云端保存，无手动保存操作；

3. 双主题：浅色办公 / 深色夜间创作；

4. 权限管控

    - 负责人：编辑全部节拍、全局校验、批量操作、锁定画布；

    - 普通编剧：仅编辑自身分配分支的节拍卡片，其他分支只读；

5. 全局检索：顶部搜索可检索节拍梗概、人物标签，画布自动定位对应卡片。
