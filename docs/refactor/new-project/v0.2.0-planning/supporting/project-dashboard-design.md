# Web 端首页设计

## 一、两种状态完整页面方案（登录后首页 = 项目管理工作台）

### 状态 1：账号存在项目（有创作记录，面向存量用户 / 负责人）

1. 页面主体：项目卡片列表布局

    - 卡片信息：项目名称、作品类型标签（线性网文 / 多分支游戏剧本）、上次编辑时间、负责人、缩略预览图（取矩阵时序画布截图）；

    - 卡片快捷操作：打开项目、复制项目、导出全量大纲、删除项目；

    - 支持筛选：按线性 / 多分支、创建时间、负责人分组检索；

2. 右上角固定悬浮按钮：【新建项目】（高强调色，视觉层级最高）；

3. 辅助元素：顶部导航「项目 / 世界观模板库 / 帮助中心」，侧边可切换个人 / 团队项目；

4. 设计优势：老用户优先看到历史项目，快速续稿，新建作为次要操作入口，不干扰核心使用流程。

### 状态 2：账号无任何项目（新注册空白账号）

1. 整体居中空白引导页，无项目列表；

2. 上层轻量化引导文案（分层，降低新手迷茫）

    - 主标题：开始创作你的小说 / 互动剧本

    - 副标题：内置线性影视、多分支游戏两套标准化创作模板，一键生成标准化创作画布；

    - 辅助小字：完善世界观→搭建时序 / 分支→细化节拍，一站式 AI 生成完整文稿；

3. 页面正中央大号主按钮【新建项目】，视觉突出；

4. 下方补充浅灰色辅助引导卡片：简单介绍两套创作流程，附带简短示意图（世界观→时序→节拍 / 世界观→分支树→时序→节拍）；

5. 无多余功能干扰，仅保留顶部基础导航，弱化复杂功能，聚焦「新建项目」唯一行动点。

## 二、适配你产品 V1\.0 核心能力的加分设计

1. 点击【新建项目】后，直接弹出模板选择弹窗（两套核心模板）

    - 模板 1：线性单结局（影视 / 网文 / 短剧，自动隐藏分支树视图）

    - 模板 2：多分支多结局（游戏 / 互动小说，完整开放分支树）
    选择模板后自动初始化配套画布预设、轨道模板、世界观表单；

2. 空白页引导文案预埋两套评审流程说明，提前给中初级用户建立创作认知，降低后续画布学习成本；

3. 项目卡片区分项目类型标签，负责人打开首页可快速区分线性项目、多分支项目，提升验收效率。

## 三、交互 \& 体验优化细节（规避常见痛点）

1. 按钮统一规范：

    - 有项目页：右上角小尺寸高亮新建按钮，不抢占页面主体；

    - 空白无项目页：居中大号主按钮，页面唯一核心操作，降低用户找不到入口的流失；

2. 状态平滑切换：用户首次新建项目后，页面自动从空白引导页刷新为项目列表页，无生硬跳转；

3. 轻量化引导：空白页不堆砌长篇教程，仅核心流程简介，复杂分步引导进入项目内部后再分步弹窗；

4. 团队账号适配：负责人首页额外增加「团队共享项目」分组，普通编剧仅展示自身分配项目。

## 四、对比竞品设计优势

1. Campfire Write、Plottr 登录后默认空白画布，无项目管理首页，多项目切换繁琐；

2. Narratium 缺少模板前置选择，新手进入后不知道如何搭建线性 / 分支故事；
你的方案通过「项目列表首页 \+ 空白引导页 \+ 新建模板弹窗」三层结构，解决多项目管理、新手入门两大痛点，同时贴合负责人批量验收、多项目统筹的使用场景。

## 五、精简落地总结

该登录首页双状态设计逻辑通顺、开发成本低、用户体验分层清晰：

3. 新建项目弹窗绑定两套标准化创作模板，统一约束 V1\.0 仅支持的两套评审 / 创作链路，产品整体逻辑闭环。
