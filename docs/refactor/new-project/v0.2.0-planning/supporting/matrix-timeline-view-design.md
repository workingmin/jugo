# Web端「矩阵时序视图」页面完整设计方案（对标Plottr，适配V1\.0双评审流程、Web轻量化、与世界观/分支树/节拍板数据互通）

## 一、基础定位

1. 载体：纯 Web 页面，PC 端为主，支持宽屏横向拖拽，适配编剧长时序排布；

2. 对标基准：Plottr 横轴统一时间轴 \+ 纵轴分轨道并行支线矩阵结构；

3. 差异化升级（弥补 Plottr 短板）：

    - 与世界观知识库双向联动；

    - 与分支树视图联动筛选单分支时序；

    - 卡片一键跳转节拍板；

    - 内置全局设定冲突一键校验；

    - 适配负责人自上而下评审流程；

4. 核心使用链路：
线性项目：世界观 → 矩阵时序 → 节拍板
多分支项目：世界观 → 分支树选中单分支 → 过滤后进入本视图查看专属时序

## 二、页面整体布局（四栏 Web 分层布局）

### 1\. 顶部全局操作栏（固定置顶）

通用全局控件 \+ 视图快捷切换，统一产品导航规范

1. 视图切换组：世界观 / 矩阵时序 / 分支树 / 节拍板（一键跳转）

2. 筛选控制区

    - 分支过滤器（多分支项目显示）：下拉选择单分支 / 全部分支，自动隐藏无关轨道卡片；线性项目隐藏该控件

    - 角色 / 势力标签筛选：勾选仅展示对应人物、势力相关剧情卡片

3. 画布操作工具

    - 缩放滑块：50%–200% 画布缩放

    - 自动排布：一键规整所有卡片对齐时间轴

    - 全局校验：扫描时序、人设、伏笔冲突，生成验收报告

    - 批量导出：导出当前时序大纲

4. 画布设置

    - 新增纵向轨道、重命名轨道、轨道颜色自定义

    - 时间刻度显示开关（章节 / 集数 / 页码刻度）

5. 帮助引导：匹配当前项目模板（线性 / 多分支）分步指引

### 2\. 左侧轨道管理侧边栏（纵向轨道控制面板，对标 Plottr 纵轴支线）

1. 轨道列表 = 纵轴每一条并行故事线（主线、女主线、反派线、配角线、势力线等）

2. 每条轨道配置项：

    - 轨道名称、自定义色块（卡片同步同色，快速区分支线）

    - 显隐开关：临时隐藏本条轨道，简化评审视图

    - 关联世界观：绑定对应人物 / 势力，筛选时自动联动

    - 右键菜单：重命名、删除、复制轨道、批量生成节拍卡片

3. 底部操作：【新建轨道】、【系统模板轨道一键导入】（三幕主线 / 男女主 / 反派预设轨道）

### 3\. 中央核心画布区（矩阵时序主体，对标 Plottr 画布）

#### 画布底层结构

- **横轴：统一全局线性时间刻度**
刻度可自定义单位：章节、剧集、剧本页码、剧情阶段；从左到右代表故事推进顺序；

- **纵轴：左侧侧边栏定义的独立轨道**，横向铺满画布；

- 单元格交点：剧情卡片可拖拽放置在对应轨道 \+ 对应时间刻度坐标；

- 视觉规则：同一时间横轴坐标下，多条轨道同时存在卡片 = 并行同步发生的多条支线；无分叉、无互斥选择（区分分支树）。

#### 剧情卡片（核心交互单元）

1. 卡片基础信息

    - 标题：场景核心情节点；

    - 缩略梗概：1–2 行简介；

    - 色块：跟随所属轨道配色；

    - 标签：出场人物、势力、伏笔、分支归属（多分支项目）；

2. 卡片交互能力（贴合评审流程）

    - 拖拽：横向拖动改变发生时间，纵向拖动切换所属轨道；

    - 双击：快速编辑卡片梗概、新增标签；

    - 右键菜单：
    ① 跳转节拍板：直接打开本场完整场景细化页面（负责人评审核心操作）
    ② 关联世界观：快速绑定人物 / 地点 / 伏笔，同步至知识库
    ③ 添加评审批注：红点标记，全视图同步可见
    ④ 复制卡片、删除卡片、批量复制到其他分支

3. 分支标记（多分支项目专属）
属于分叉节点的卡片左上角小三角标记，鼠标悬浮可跳转分支树对应分叉点位；线性项目无标记。

#### 画布辅助元素

1. 纵向分区线：划分三幕 / 四幕结构，可拖拽调整幕长，对标好莱坞节拍体系；

2. 伏笔标记：绑定伏笔库的卡片底部显示虚线标识；

3. 空白间隙高亮：自动识别某条轨道长时间无卡片，提示支线断层（负责人快速识别结构缺陷）。

### 4\. 右侧详情面板（选中卡片后弹出，自动吸附）

选中任意时序卡片，右侧加载详情，无需新开页面：

1. 卡片完整梗概编辑区；

2. 关联标签：人物、势力、地点、伏笔，点击标签直接跳转世界观知识库；

3. 关联分支（多分支项目）：显示该卡片归属哪几条结局分支；

4. 评审批注列表：所有负责人 / 组员修改意见；

5. 快捷按钮：【进入节拍板细化本场】【同步更新至世界观】。

## 三、线性项目 vs 多分支项目 视图差异化控制

### 1\. 线性单结局项目（模板 1）

- 隐藏「分支筛选器」、分支标记、分支归属字段；

- 画布仅展示统一主线并行轨道，无分叉相关 UI；

- 评审流程：世界观→本画布→节拍板。

### 2\. 多分支互动项目（模板 2/3/4）

- 顶部开启分支筛选下拉；

- 卡片显示分支归属标签、分叉三角标记；

- 从分支树选中某一条结局分支后，本视图自动过滤，仅展示该分支全部轨道卡片，其余分支卡片灰化隐藏；

- 评审流程：世界观→分支树筛选单分支→本画布查看专属时序→节拍板。

## 四、核心联动能力（超越原生 Plottr，V1\.0 硬性需求）

### 1\. 与世界观知识库双向互通

1. 画布卡片绑定人物 / 势力 / 伏笔 → 自动同步标签至知识库对应条目；

2. 知识库修改人设、势力关系后，画布全局校验并标记冲突卡片；

3. 卡片标签点击，一键跳转世界观详情页。

### 2\. 与分支树视图联动

1. 分支树选中分支 → 时序画布自动过滤，仅展示该分支内容；

2. 时序上分叉节点卡片，可一键跳转分支树查看分叉选择逻辑；

3. 修改分支树分叉时间点，时序画布卡片横轴坐标自动同步调整。

### 3\. 与节拍板双向互通

1. 时序卡片一键跳转节拍板，编辑场景细节；

2. 节拍板修改场景梗概，时序卡片缩略文字实时刷新；

3. 批量操作：多条时序卡片批量生成对应节拍条目。

### 4\. 全局冲突校验（负责人验收核心功能）

点击顶部「全局校验」，自动扫描：

1. 人物行为与世界观人设冲突；

2. 支线长时间断层、节奏失衡；

3. 伏笔只出现无回收；

4. 多分支时序交叉矛盾；
输出报告，每条问题卡片附带「定位画布」「跳转节拍」按钮。

## 五、适配负责人评审流程的专属交互设计

1. 极简下钻链路：分支树筛选单分支 → 进入矩阵时序全局浏览该分支完整时间线与多支线均衡度 → 发现异常卡片右键跳转节拍板核查细节；

2. 全局批注：在时序画布任意卡片添加修改意见，分支树、节拍板同步红点标记；

3. 轨道快速显隐：负责人可临时隐藏无关支线，聚焦主线 / 单条分支；

4. 间隙高亮预警：自动标出某角色长期无出场，快速识别剧情失衡硬伤。

## 六、对比 Plottr 原生产品的差异化优势

1. Plottr 短板

    - 独立客户端，无 Web 在线协作；

    - 时序画布与设定库完全割裂，无联动校验；

    - 无原生分支联动，无法过滤单分支时序；

    - 无内置 AI 配套、全局冲突检测；

    - 无法一键跳转独立场景细化页面。

2. 本 Web 矩阵时序视图优化点

    - Web 在线多人协作，负责人可实时查看组员时序；

    - 一套底层数据连通世界观、分支树、节拍板；

    - 适配多分支项目筛选过滤，贴合游戏 / 互动剧本评审；

    - 内置验收校验工具，专门面向项目负责人自查；

    - 轻量化，无需下载客户端，浏览器直接打开。

## 七、视觉与交互规范（Web 端）

1. 画布支持拖拽平移、滚轮缩放，操作逻辑和 Plottr 保持一致，降低用户学习成本；

2. 全部修改自动云端保存；

3. 双主题：浅色（白天办公）、深色（夜间创作）；

4. 权限区分：

    - 负责人：可编辑所有轨道、所有分支、全局校验；

    - 普通编剧：仅编辑分配给自己分支的轨道卡片，其他分支只读；

5. 全局搜索：顶部搜索框可检索时序卡片文字、标签，画布自动定位卡片。
