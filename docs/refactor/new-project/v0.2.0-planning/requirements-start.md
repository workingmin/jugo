# 需求分析起点

## 待确认问题

### 产品目标

- JUGO 新版本要解决的第一优先级问题是什么？
- 面向个人作者、团队编剧、网文工作室，还是其他用户？
- 新版本首个可验证成果是什么？

### MVP 范围

- 必须包含哪些创作流程？
- 小说和剧本是否仍然同等优先？
- AI 能力是核心入口、辅助能力，还是可插拔服务？
- 是否需要账号体系、作品云端存储、多端同步？
- 是否需要导出、版本历史、素材库、协作？

### 非目标

- 哪些旧功能明确不再做？
- 哪些能力延后到 `v0.3.x` 或更晚？
- 哪些第三方参考项目不再纳入新方案？

### 数据和接口

- 核心对象是作品、章节、场景、角色、素材、任务，还是需要重新抽象？
- 小说章节和剧本场景是否属于同一内容模型？
- AI 任务是同步返回、异步任务，还是 WebSocket/队列驱动？

### 工程约束

- 是否继续使用 Go + Gin + React + Vite？
- 是否继续保留当前 `backend/`、`frontend/`、`web-prototype/` 目录？
- 是否需要重新设计数据库迁移、测试、部署和配置体系？

## 首批决策建议

1. 先写 `product-scope.md`，确认 MVP 和非目标。
2. 再写 `domain-model.md`，统一小说、剧本和 AI 任务的业务对象。
3. 然后写 `api-contract-draft.md`，避免前后端再次分叉。
4. 最后再决定哪些旧代码复用。
